## Quick Start

### 1. Set up admin credentials (do this first)
```bash
cd server
npm install
node generate-hash.js yourChosenPassword
```
Copy the output hash into `server/.env` as `ADMIN_PASSWORD_HASH`.

Generate a JWT secret:
```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```
Copy the output into `server/.env` as `JWT_SECRET`.

### 2. Start the backend
```bash
cd server
npm run dev
```

### 3. Start the frontend
```bash
cd client
npm install
npm run dev
```

- Store: http://localhost:5173
- Admin login: http://localhost:5173/admin/login
- API health: http://localhost:5000/api/health

## Environment Variables

## Security Setup

### Admin credentials
- Username is set via `ADMIN_USERNAME` in server/.env (default: admin)
- Password is bcrypt-hashed — never stored in plain text
- JWT expires in 7 days by default (set `JWT_EXPIRES_IN` to change)
- All write routes (POST/PUT/DELETE) require a valid JWT
- MongoDB operator injection blocked by express-mongo-sanitize
- Security headers set by helmet on every response
- gzip compression enabled on all responses

## MongoDB — Restricted User (Required before going live)

The default Atlas connection string has full admin rights.
Create a restricted user that can only access your store database:

1. Atlas Dashboard → Database Access → Add New Database User
2. Choose "Password" authentication
3. Under "Database User Privileges" → select "Custom Role"
4. Click "Add Built In Role" → choose "readWrite"
5. Database: your store database name (e.g. mystore)
6. Leave Collection blank (access all collections in that DB only)
7. Click "Add User"

Update MONGO_URI in server/.env to use the new restricted user credentials.

This user cannot drop databases, access other clusters, or create new databases.
If the URI ever leaks, the damage is limited to your store data only.
